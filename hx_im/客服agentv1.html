步骤模块能力说明性能
Step 0memory_manager长期记忆回忆新会话首轮，根据 user_id 从 ChromaDB 召回用户画像 + 历史会话摘要ChromaDB
Step 0.5text_corrector文本纠正 NEW三级策略：上下文正则 → 30+ 词领域词典 → LLM 兜底（仅可疑文本触发）。解决 "吃出所料" → "吃出塑料" 等同音字问题<1ms 词典
~150ms LLM
Step 1intent_classifier两级意图识别Tier-1 向量余弦相似度（20+ 意图原型）→ Tier-2 LLM CoT 推理兜底。支持多意图检测<50ms T1
~150ms T2
Step 1.5dialogue_manager话题切换与书签LLM 判断 none / temporary / permanent。临时切换保存书签（意图+槽位+确认状态），回归时自动恢复1次 LLM
Step 2stage_detector五信号阶段融合意图映射(35%) + 业务数据(25%) + 实体信号(15%) + 关键词(15%) + 上下文惯性(10%)<1ms

Step 6slot_extractor混合槽位提取正则（订单号/手机号）+ LLM 语义（菜品/原因/地址/数量）。按意图定义必要槽位1次 LLM
Step 8retriever + KB意图路由 RAG按意图选择集合（products/faq/policies），语义搜索 top-3，去重 + 过滤(≥0.55)ChromaDB
Step 8.5function_executor函数调用系统8 个业务函数（查询3 + 操作5），声明式注册，二次确认流程，auto_query 自动触发1次 LLM

Step 9generator回复生成SOP 指令 + 记忆上下文 + RAG 知识 + 函数结果 → 全量注入 LLM 生成自然语言回复1次 LLM

三、业务函数清单

函数名类型需确认绑定意图说明
query_order查询否全部售后意图查询订单详情（状态、商品、金额）
query_delivery_status查询否delivery_inquiry, order_rush查询骑手位置、预计到达时间
query_insurance_status查询 auto否food_safety自动触发：order_id 满足即查询保险状态
cancel_order操作是order_cancel取消订单，需用户二次确认
submit_refund操作是refund_request, wrong_item, missing_item ...提交退款申请
modify_order操作是order_modify加菜、改备注、改地址
rush_order操作否order_rush催单，通知商家/骑手
submit_complaint操作是complaint提交投诉工单
