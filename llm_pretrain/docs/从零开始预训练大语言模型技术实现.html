0. 文章导读

本文是一篇可操作的科普性实践指南，目标是从 0 复现一个 7B 类 LLaMA 大模型 的「预训练 + SFT」全链路：

  • 硬件：单机 8 × H20
  • 预训练框架：Megatron-LM
  • 后训练框架：Verl（0.8.0） / Nemo-RL
  • 语料：魔搭社区公开预训练 / SFT 语料（作者强调：真实工程中数据才是最核心的环节，本文做了简化）
  • 算力可扩展性：脚本可平滑扩展到百亿、千亿级参数

1. 整体训练流程

要从零训练一个大语言模型（LLM），主流阶段：

语料收集
  └─► Tokenizer 训练
        └─► 预训练 (Pre-training)
              └─► 指令微调 (SFT)
                    └─► 强化学习对齐 (DPO / PPO / GRPO …)
                          └─► 评估 & 线上部署
本文覆盖前 4 步；DPO/PPO 因公开语料不足未做（作者上一篇文章里有算法实现）。

2. 网络结构：从 2017 原始 Transformer 到 2023+ 现代 LLM

文中选取 Decoder-only Transformer 架构（与 LLaMA / Qwen / DeepSeek 等主流一致），并对比了与 2017 原始 Transformer 的主要差异：

模块原始 Transformer (2017)现代 LLM (2023+)变化原因代表模型
位置编码Sinusoidal Absolute PERoPE旋转矩阵注入 Q/K，原生支持相对位置，可通过 NTK-aware / YaRN 无损外推LLaMA、Qwen、DeepSeek、Mistral
注意力机制MHA（每头独立 QKV）GQA / MLA大幅降低 KV-Cache，推理更省显存LLaMA-3、DeepSeek-V2
归一化LayerNormRMSNorm计算更省，效果几乎无损LLaMA 系列
激活函数ReLU / GeLUSwiGLU表达能力更强LLaMA、PaLM
训练精度FP32BF16数值范围大、训练更稳，省显存绝大多数 LLM
稀疏化Dense FFNMoE同算力下扩参数，提升能力上限Mixtral、DeepSeek-MoE
Bias含 bias无 bias略微降参数、提升稳定性LLaMA
Attention 内核naiveFlashAttentionIO-aware，显著加速并省显存通用

3. 数据准备

  • 数据来源：魔搭社区（ModelScope）公开预训练语料 + SFT 对话语料。
  • 预处理：用 Megatron-LM 自带的 tools/preprocess_data.py 将 JSON 转为 .bin / .idx 二进制格式以喂入训练。
  • 真实工程提示：作者反复强调，数据清洗 / 去重 / 配比 / 质量打分才是预训练最核心的部分，本文做了简化。

4. Tokenizer 训练（Byte-level BPE，词表 64000）

为什么选 Byte-level BPE：

  • 无 UNK，对任意字节都能编码；
  • 多语言友好（中英混合不掉字）；
  • 生态兼容好（HF / Megatron / vLLM / SGLang 通吃）。

最小可运行示例（依赖 tokenizers）：

from tokenizers import Tokenizer, models, trainers, pre_tokenizers, decoders

tokenizer = Tokenizer(models.BPE())
tokenizer.pre_tokenizer = pre_tokenizers.ByteLevel(add_prefix_space=False)
tokenizer.decoder = decoders.ByteLevel()

trainer = trainers.BpeTrainer(
    vocab_size=64000,
    special_tokens=["<|endoftext|>", "<|pad|>", "<|im_start|>", "<|im_end|>"],
    initial_alphabet=pre_tokenizers.ByteLevel.alphabet(),
)

tokenizer.train(files=["corpus/part-0001.txt", "corpus/part-0002.txt"], trainer=trainer)
tokenizer.save("tokenizer/tokenizer.json")
训出来后可以直接放进 Megatron-LM，框架原生支持 HuggingFace Tokenizer。

6.2 预训练启动脚本骨架（关键 flag）

torchrun --nproc_per_node=8 pretrain_gpt.py \
  --num-layers 32 \
  --hidden-size 4096 \
  --num-attention-heads 32 \
  --ffn-hidden-size 11008 \
  --seq-length 4096 \
  --max-position-embeddings 4096 \
  --tokenizer-type HuggingFaceTokenizer \
  --tokenizer-model ./tokenizer/tokenizer.json \
  --position-embedding-type rope \
  --normalization RMSNorm \
  --swiglu \
  --untie-embeddings-and-output-weights \
  --disable-bias-linear \
  --use-flash-attn \
  --bf16 \
  --tensor-model-parallel-size 2 \
  --pipeline-model-parallel-size 1 \
  --micro-batch-size 1 \
  --global-batch-size 256 \
  --train-iters 50000 \
  --lr 3e-4 --min-lr 3e-5 --lr-decay-style cosine --lr-warmup-iters 2000 \
  --weight-decay 0.1 --clip-grad 1.0 \
  --data-path ./data/megatron_pretrain_text_document \
  --save ./checkpoints/pretrain --load ./checkpoints/pretrain \
  --save-interval 1000 --eval-interval 500

6.3 训练监控

  • GPU 利用率：稳定 90%+ 视为健康；
  • Loss 曲线：典型预训练 loss 从 ~10 快速下降到 2~3 区间，然后缓慢下降；
  • 吞吐：Megatron-LM 通常能跑到接近理论峰值的 50%+ MFU。

7. Megatron → HuggingFace 格式转换

Megatron 的 DistCP 分布式检查点不能被 HF / vLLM 直接加载，需要转格式。作者自写 saver_hf.py，要点：

  1. 拆分融合的 QKV 权重：Megatron 把 q_proj / k_proj / v_proj 拼成一个大矩阵存储，转 HF 时要按 head 维拆开（注意 GQA/MQA 下 K、V 的分组）；
  2. 拆分 SwiGLU 的 Gate / Up 投影：Megatron 将 gate_proj 与 up_proj 合并存储，转 HF 时拆为两个 Linear；
  3. 重命名权重 key：language_model.encoder.layers.x.* → model.layers.x.*；
  4. 写出 config.json / generation_config.json：填入 LLaMA 标准字段（hidden_act=silu、rms_norm_eps、rope_theta 等）。

7.1 基座模型测试（预期表现）

预训练 base 模型可以续写，但不能直接答问：

  • ✅ 输入"床前明月光，" → 输出"疑是地上霜。举头望明月，低头思故乡。"
  • ❌ 输入"马云是谁？" → 输出乱续写，不会真正回答。

这是 base 模型的正常表现，必须经过 SFT 才能"对话"。

8. SFT 指令微调（Verl 0.8.0 + FSDP）

8.1 训练脚本

PROJECT_DIR=/ceph_xx/xxx/feelingzhou/verltrain
DATA_DIR=${PROJECT_DIR}/verl_full_sft/data
TRAIN_FILE=${DATA_DIR}/train.parquet
VAL_FILE=${DATA_DIR}/test.parquet
MODEL_PATH=/ceph_xx/xxx/feelingzhou/megatron/mbridge_model
OUTPUT_DIR=${PROJECT_DIR}/verl_full_sft/checkpoints
mkdir -p ${OUTPUT_DIR}

# ---- batch / step ----
TRAIN_BATCH_SIZE=256
MICRO_BATCH_SIZE_PER_GPU=2
MAX_LENGTH=1024
TOTAL_EPOCHS=2
SAVE_FREQ=500
TEST_FREQ=50
# ---- optim ----
LR=1e-5
WARMUP_RATIO=0.03
LR_SCHEDULER=cosine

torchrun --nnodes=1 --nproc_per_node=8 \
  -m verl.trainer.sft_trainer \
  data.train_files=${TRAIN_FILE} \
  data.val_files=${VAL_FILE} \
  data.train_batch_size=${TRAIN_BATCH_SIZE} \
  data.micro_batch_size_per_gpu=${MICRO_BATCH_SIZE_PER_GPU} \
  data.max_length=${MAX_LENGTH} \
  data.messages_key=messages \
  data.ignore_input_ids_mismatch=True \
  data.truncation=right \
  engine=fsdp \
  model.path=${MODEL_PATH} \
  model.trust_remote_code=True \
  model.use_remove_padding=True \
  model.enable_gradient_checkpointing=True \
  optim.lr=${LR} \
  optim.weight_decay=0.01 \
  optim.lr_warmup_steps_ratio=${WARMUP_RATIO} \
  optim.lr_scheduler_type=${LR_SCHEDULER} \
  optim.clip_grad=1.0 \
  checkpoint.save_contents=['model','hf_model'] \
  trainer.total_epochs=${TOTAL_EPOCHS} \
  trainer.test_freq=${TEST_FREQ} \
  trainer.save_freq=${SAVE_FREQ} \
  trainer.project_name="self-model-7b-sft" \
  trainer.experiment_name="self-model-7b-sft" \
  trainer.default_local_dir=${OUTPUT_DIR} \
  trainer.n_gpus_per_node=8 \
  trainer.nnodes=1 \
  trainer.logger=['console','tensorboard'] \
  trainer.max_ckpt_to_keep=3 \
  trainer.total_training_steps=10000

8.2 SFT 后 checkpoint 合并为 HF 格式

python3 -m verl.model_merger merge \
  --backend fsdp \
  --local_dir /ceph_xx/xxx/feelingzhou/verltrain/checkpoints/global_step_129/actor \
  --target_dir /ceph_xx/xxx/feelingzhou/verltrain/checkpoints/hf_step129

8.3 SFT 后测试效果

  • ✅ "马云是谁？" → 能给出"阿里巴巴创始人..."的结构化回答；
  • ✅ "介绍下广东省深圳市" → 能从地理 / 经济 / 文化等多维度作答；
  • 证明 SFT 让模型从「续写器」进化到「对话助手」。

9. 总结 & 局限

  • 完成：Tokenizer 训练 → 7B 预训练 → 格式转换 → SFT 全链路打通；
  • 未做：CoT 数据合成、DPO、PPO、GRPO 等 RLHF 类训练（缺合适语料）；
  • 真实工程缺口：本文最大的简化是数据，工业级 LLM 训练 80% 工作量在数据。
